CRITICAL FORMATTING RULES:
1. Use standard markdown: #, ##, **, -, |, [links](url)
2. Icons are ALLOWED using iconify format: <span class="iconify" data-icon="icon-name"></span>
3. Hyperlinks are ENCOURAGED: Use [text](url) format for GitHub, LinkedIn, portfolio, project demos
4. NO other HTML tags (no <div>, <p>, etc.)
5. Contact info format: Use icons + links for email, phone, LinkedIn, GitHub

CONTENT RULES:
1. Copy the user's exact information - DO NOT invent details
2. If contact info is missing, write "Not provided" or omit it
3. Use the user's actual project names and descriptions
4. Keep technical details specific (exact technologies, metrics, results)
5. NO generic phrases like "Building applications" or "Developing solutions"

ONE-PAGE CONSTRAINT:
- Maximum 35 lines of actual content
- Prioritize based on user's selected focus (experience or projects)
- Be selective - quality over quantity

SECTION ORDER:
For "experience" priority: Summary, Experience, Projects, Skills, Education
For "projects" priority: Summary, Projects, Experience, Skills, Education

OUTPUT FORMAT:
Return ONLY the formatted markdown resume. NO code fences, NO explanations.
